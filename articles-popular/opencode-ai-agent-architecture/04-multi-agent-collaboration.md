# 第四篇: 多智能体协作机制

> **用人话说**：一个 Agent 干不过来的时候，可以派多个 Agent 一起干。怎么分工、怎么协作、怎么汇总结果，就是这篇要讲的。

---

## Orchestrator-Worker 模式

OpenCode 采用经典的 Orchestrator-Worker（指挥官-工人）模式：

```
用户 → 主 Agent (build/plan) → Subagent A / Subagent B / Tool Calls
```

用人话说：
- 主 Agent 是"项目经理"，负责接收任务、分配任务、汇总结果
- Subagent 是"工程师"，负责具体执行
- 工具是"螺丝刀"，Agent 用工具干活

---

## 四种协作模式

### 模式一：并行探索

同时派出多个 Agent 搜索不同方向：

```
主 Agent 同时派：
  → explore Agent A：搜索认证相关代码
  → explore Agent B：搜索错误处理相关代码
  → explore Agent C：搜索配置相关代码

主 Agent 继续干活，完成时收到通知
```

**用人话说**：就像项目经理同时派 3 个人去查 3 个不同的问题，自己继续干别的活。

### 模式二：串行委派

分步骤执行，前一步的结果喂给下一步：

```
主 Agent 派 general Agent：
  → 第一步：研究需求
  → 第二步：基于研究结果写代码
  → 第三步：基于代码写测试
```

**用人话说**：就像流水线，上一道工序的产出是下一道工序的原料。

### 模式三：会话延续

保留上下文继续对话：

```
主 Agent 派 deep Agent 分析：
  → 第一次：分析架构问题
  → 第二次：继续分析（保留之前的上下文）
  → 第三次：继续分析（还是同一个会话）
```

**用人话说**：就像和同一个人连续开会，每次都记得之前讨论过什么。

### 模式四：工具共享

Agent 通过工具系统协作：

```
Agent A 写了一个文件
Agent B 读取这个文件并修改
Agent C 读取修改后的文件并测试
```

**用人话说**：就像多人协作写文档，通过共享文件夹传递成果。

## SessionRunCoordinator：会话协调器

协调器管理会话的执行流：

| 方法 | 功能 | 用人话说 |
|---|---|---|
| resume | 启动或加入会话 | 开始干活 |
| wake | 唔醒会话 | 把休眠的会话叫醒 |
| interrupt | 中断会话 | 停止干活 |

**关键规则**：
- 同一 Session 串行执行（不能同时干两件事）
- 不同 Session 并行执行（可以同时干不同的事）

**用人话说**：就像一个人不能同时做两件事，但多个人可以同时做不同的事。

---

## Eager Tool Execution：抢先执行

V2 运行器的特性：解析到工具调用就立即执行，不等 LLM 流结束。

```
LLM 流 → 解析到 tool_call → 立即执行
       → 继续解析流
LLM 流结束 → 等待所有工具 → 持久化 → 判断继续
```

**用人话说**：就像工厂的流水线，零件一到就立即加工，不等整条线停了再一起处理。这样能减少延迟。

---

## BackgroundJob 系统

管理后台任务的生命周期：

```
start() → running → extend() → completed/error/cancelled
```

| 方法 | 功能 | 用人话说 |
|---|---|---|
| start/extend | 启动/延续任务 | 开始/继续干活 |
| wait/cancel | 等待/取消 | 等结果/不干了 |
| promote | 提升到前台 | 从后台转到前台 |
| list/get | 列表/查询 | 查看任务状态 |

**特点**：
- 非持久化（内存管理，重启就没了）
- 每个任务有独立 Effect Scope（资源隔离）
- 链式延续支持（可以接力干）

---

## 一句话总结

多 Agent 协作的核心是"分工+协调"：主 Agent 分配任务，Subagent 执行，通过会话协调器管理执行流，支持并行、串行、延续等多种模式。
